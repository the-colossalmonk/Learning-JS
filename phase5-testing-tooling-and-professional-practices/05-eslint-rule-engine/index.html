    What is a Linter?

    A "linter" is a tool that analyzes your code for potential errors, style inconsistencies, and bad practices **without actually running it**. This is called static code analysis.

    How Does It Work?

    1. Parsing: The linter reads your code and turns it into a structured data format called an Abstract Syntax Tree (AST). This tree represents the code's structure, like a grammatical breakdown of a sentence.

    2. Traversing: It then "walks" through this tree, visiting each part of your code (variables, functions, loops, etc.).

    3. Applying Rules: At each step, it checks the code against a set of predefined rules. If it finds code that breaks a rule (like using `var` instead of `let`), it reports an error.

    Why is it Useful?

    • ✅ Catches common bugs and typos before you run your code.
    • ✅ Enforces a consistent coding style across a team.
    • ✅ Teaches best practices and helps avoid problematic language features.
